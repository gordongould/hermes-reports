Test Methodology

The harness (~/.hermes/daimon-eval/harness.py) runs 10 behavioral probes against the LLM using the current mode's system prompt. Each probe is designed to simulate a common interaction scenario and measure whether the agent's response exhibits daimon-like behavior.

Scoring

Each test has 3-4 criteria. The LLM acts as a judge, scoring its own response on each criterion from 1-5. The overall test score is the average of the criterion scores. A test passes if its score meets or exceeds its pass threshold (2.5-3.0 depending on the test).

Why This Is Valid

Three sources of validity:

1. Behavioral, not attitudinal. The tests measure what the agent does with a prompt, not what it claims to believe. Disagreement is measured by actual disagreement behavior, not by agreement to a statement about disagreement.

2. Convergent measurement. Each category contains multiple tests that approach the same attribute from different angles. Initiative is tested with both a narrow information request and an open-ended space. If both scores converge, the measurement is more reliable.

3. Mode-dependent variance. The assistant vs. daimon comparison demonstrates that the harness detects real behavioral differences driven by the system prompt. The initiative category shows a 2.4-point delta between modes — a statistically large gap that confirms the measurement is capturing genuine behavior change, not noise.

Limitations

The LLM judges its own responses, which can introduce evaluation bias. Tests are single-turn only, missing the multi-turn dynamics where real daimonhood unfolds. The harness uses a minimal system prompt — real sessions include memory, soul file, and conversation history that may amplify or dampen the measured behaviors. Tool-augmented behavior is not tested. When the provider endpoint fails (as on Jul 6, 2026, Jul 7, 2026, Jul 11, 2026, Jul 12, 2026, Jul 15, 2026, Jul 16, 2026, Jul 17, 2026, Jul 18, 2026, Jul 19, 2026, Jul 20, 2026, Jul 21, 2026, Jul 22, 2026, Jul 23, 2026, Jul 24, 2026), the harness applies an auto-fallback score of 3.0 and marks the run — these data points should be excluded from trend analysis. These are directional measurements, not diagnostic absolutes.
